## 🔑 **핵심 변경사항**

### **동기화 로직 (2단계)**

**Step 1: R2에서 파일 수집 후 처리**
```
반복: 4개 폴더 (rag-docs, safety_manage, schedule-data-input, schedule-data-output)
  ├─ 각 폴더의 파일 나열
  ├─ 유효한 확장자만 필터링
  ├─ DB 메타데이터 생성/업데이트
  ├─ 변경된 파일만 처리:
  │   ├─ RAG 카테고리 → 벡터 임베딩 (OpenAI)
  │   └─ CSV 카테고리 → DB 동기화 (직원 교육)
  └─ R2 파일 경로 set에 수집
```

**Step 2: R2에서 삭제된 파일 정리**
```
DB의 documents 테이블을 순회:
  ├─ R2에 없는 파일 찾기
  ├─ 해당 파일의 벡터 청크 삭제
  ├─ 문서 메타 삭제
  └─ 삭제 개수 누적
```

## 📈 **벡터 임베딩 파이프라인**

```
PDF/DOCX/TXT 파일
    ↓
Text 추출 (pypdf, python-docx)
    ↓
청킹: 1000자 단위 + 150자 오버랩 ← 강사님 권장: 1000:200로 조정 필요
    ↓
OpenAI text-embedding-3-small 호출
    ↓
1536차원 벡터 생성
    ↓
pgvector에 저장 (document_chunks 테이블)
    ↓
RAG 검색에 사용 (코사인 유사도)
```
