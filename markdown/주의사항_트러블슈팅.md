## 🐛 **주요 에러 및 해결책**

### **에러 1: ForeignKeyViolation (emp_id 불일치)**

```
ERROR: insert or update on table 'safety_training' violates foreign key constraint
Key (emp_id)=(EMP001) is not present in table 'employees'
```

**원인:** CSV의 emp_id가 대문자, DB는 소문자

**해결책:** 이미 수정됨 (`csv_sync_service.py` 라인 283)
```python
emp_id = row.get(emp_key, "").strip().lower()  # ← 소문자 변환
```

**확인:** 
```bash
# CSV 업로드 전 emp_id 형식 확인
# EMP001, emp001, 사번 등 섞여있으면 안 됨
```

---

### **에러 2: TRUNCATE TABLE 외래키 제약**

```
ERROR: cannot truncate a table referenced in a foreign key constraint
```

**원인:** 외래키가 있는 테이블을 TRUNCATE하려고 함

**해결책:** 이미 수정됨 (`csv_sync_service.py` 라인 80)
```python
db.execute(text(f"TRUNCATE TABLE {quoted} CASCADE"))
# ← CASCADE 추가로 외래키 무시
```

### **에러 4: 임베딩 벡터 저장 실패**

```
ERROR: pgvector type mismatch
```

**원인:** 벡터 차원 불일치
- 기존: 1536차원 (text-embedding-3-small)
- 새로운 모델: 3072차원 (text-embedding-3-large)

**해결책:**
```python
# config.py에서 절대 변경 금지
OPENAI_EMBEDDING_MODEL: str = "text-embedding-3-small"  # ← 고정!
```

---

### **에러 5: OpenAI API Key 오류**

```
ERROR: OPENAI_API_KEY is not set
```

**원인:** `.env`에 OPENAI_API_KEY 없음

**해결책:**
```bash
# .env 파일 확인
OPENAI_API_KEY=sk-proj-...  ← 실제 키 입력

# 재로드
docker compose restart backend
```

### **에러 6: 타임아웃 (특히 큰 파일)**

```
Request timeout after 30s
```

**원인:** 
- 파일이 너무 큼 (200MB 초과)
- 네트워크 느림

**해결책:**

```python
MAX_UPLOAD_SIZE_MB: int = 200  # 현재값

# API 요청 타임아웃 설정 (필요시)
# backend/app/services/embedding_service.py에서 OpenAI 클라이언트 타임아웃 조정
```

## 📝 **CSV 업로드 시 주의사항**

### **직원교육이력.csv 포맷**

**필수 헤더:**
```
사원ID (또는 emp_id, 사번) | 교육명 | 교육명 이수일 | 교육명 만료일
```

**예시:**
```
emp_id | 안전교육 | 안전교육 이수일 | 안전교육 만료일 | 역량개발 | 역량개발 이수일 | 역량개발 만료일
emp001 | Y | 2025-07-15 | 2027-07-15 | Y | 2025-08-01 | 2027-08-01
emp002 | Y | 2025-08-14 | 2027-08-14 | N | | 
```

**주의사항:**
```
1. emp_id는 반드시 소문자 (emp001, emp002 ...)
2. 날짜 형식: YYYY-MM-DD
3. 콤마로 구분 (UTF-8 인코딩)
4. 빈 값은 그냥 두거나 NULL
```

## 🔍 **검증 쿼리**

### **1. 동기화 확인**
```sql
-- safety_manage 폴더의 모든 파일
SELECT file_name, file_path, file_size 
FROM documents 
WHERE file_path LIKE '%safety_manage%'
ORDER BY file_name;

-- 임베딩 완료 파일 수
SELECT COUNT(*) as completed 
FROM documents 
WHERE embedding_status = 'completed';

-- 실패한 파일
SELECT file_name, embedding_status
FROM documents
WHERE embedding_status = 'failed';
```

### **2. 벡터 확인**
```sql
-- 저장된 청크 개수
SELECT COUNT(*) as total_chunks FROM document_chunks;

-- 특정 파일의 청크
SELECT chunk_index, content, embedding::text
FROM document_chunks
WHERE file_id IN (
  SELECT file_id FROM documents 
  WHERE file_name LIKE '%NeoChip%'
);
```

### **3. CSV 동기화 확인**
```sql
-- 직원별 교육 이력
SELECT emp_id, COUNT(*) as training_count
FROM safety_training
GROUP BY emp_id
ORDER BY emp_id;

-- 만료된 교육
SELECT emp_id, training_name, expired_date
FROM safety_training
WHERE training_status = '만료'
ORDER BY emp_id;
```
